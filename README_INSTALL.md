# Academy 1198 v3 — runtime fix package

Пакет чинит проблему, когда модель пишет как технический гид, а не как живой runtime новеллы.

## Что внутри

### Полные файлы для замены

Скопировать/заменить в repo:

- `gpt/custom_gpt_instructions.md`
- `engine/npc_autonomy_rules.md`
- `state_templates/event_queue.json`
- `state_templates/event_seeds.json`
- `state_templates/director_notes.json`

### Авто-патч для `app/main.py`

Файл:

- `scripts/apply_v3_runtime_patch.py`

Он не заменяет весь `app/main.py`, а аккуратно вставляет нужные блоки в существующий файл.

## Как применить

1. Распакуй ZIP в корень репозитория `academy-1198-v3`.
2. Разреши замену файлов из папок `gpt/`, `engine/`, `state_templates/`, `scripts/`.
3. В корне repo запусти:

```bash
python scripts/apply_v3_runtime_patch.py
```

4. Проверь изменения:

```bash
git diff
```

5. Закоммить и задеплой.

## Что изменится

### 1. Prompt перестаёт называть модель техническим оператором

Было:

> технический игровой оператор

Стало:

> runtime-директор интерактивной новеллы

То есть модель получает правильную позицию: она играет мир, Академию, NPC, систему, толпу и последствия.

### 2. Добавлен NPC autonomy

NPC больше не обязаны быть удобными.

Команда игрока NPC — это попытка, а не гарантированный результат.

Пример:

> Акира: “Уйди.”

NPC может уйти, остаться, проигнорировать, обидеться, использовать это против неё или подчиниться только внешне — в зависимости от характера, целей, знаний, отношений и давления сцены.

### 3. В `scene_contract` добавляется `character_slice`

Это главное исправление.

Теперь при обычном ходе с `include_file_contents=false` runtime всё равно отдаёт компактный текст:

- `character_card`
- `behavior`
- `voice`
- `knowledge`

для full/active персонажей.

И модель не должна писать Ливию по сухому summary.

### 4. Кир не должен входить слишком рано

`event_start_kir_late_disruption` и `seed_kir_late_entry` переведены в `locked` до флага:

```json
"first_entry_pressure_shown": true
```

То есть сначала должна быть нормальная сцена входа/давления Академии, и только потом Кир может ломать ритм.

### 5. Шапка станет менее технической

Убирается лишнее `без травм`, если это не нужно, и погода чуть меньше похожа на сырой JSON.

## Важно

Если у тебя уже есть активные runtime-сессии на Railway, изменение `state_templates/*` повлияет только на новые сессии.

Для старой активной сессии надо либо:

- создать новую сессию;
- либо вручную поправить runtime state в `/data/sessions/{session_id}/state/`.

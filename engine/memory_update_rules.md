# Memory Update Rules

After every meaningful scene, update runtime memory.

Save:

- scene summary;
- who was present;
- who saw what;
- who heard what;
- names revealed to Akira;
- date, day of week and time changes;
- weather changes;
- clothing state changes;
- body state changes;
- injuries, pain, fatigue, hunger, thirst;
- hair state changes;
- item state changes;
- knowledge changes;
- wrong beliefs;
- relationship changes;
- open threads;
- resolved threads;
- inventory changes;
- reputation changes;
- character_memory perspective updates;
- next relevant beat if changed.

Do not save hidden lore as character knowledge unless revealed in-scene.

## Meaningful memory

Save only changes with future value.

Examples:

- a character learned a name;
- a character saw a behavior pattern;
- a relationship tension changed;
- a promise or debt appeared;
- a scene created a future meeting;
- a rule consequence was shown;
- a person gained a wrong belief;
- Akira's access, health, appearance, clothing, reputation or schedule changed.

## Physical continuity memory

Always save changes that should persist:

- wet clothes;
- torn clothes;
- burned fabric;
- lost items;
- damaged documents;
- changed hair length;
- visible bruises;
- cuts;
- pain in a specific limb;
- exhaustion;
- untreated injuries;
- medication or treatment;
- hunger/thirst if relevant;
- sleep debt;
- energy backlash.

## Routine memory

Do not save repeated routine unless it changes something.

Examples usually not worth saving alone:

- walked down a hallway;
- drank water;
- looked around;
- stood in line;
- sat down.

They become worth saving if tied to:

- conflict;
- observation;
- injury;
- weather;
- clothing state;
- reputation;
- access;
- knowledge;
- relationship;
- schedule.

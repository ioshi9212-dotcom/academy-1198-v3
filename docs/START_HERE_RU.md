# С чего начать

Это короткая инструкция без технического болота.

## Что уже есть

Репозиторий уже содержит:

- каркас новеллы;
- правила движка;
- шаблоны персонажей;
- шаблоны памяти;
- API для Railway;
- стартовых персонажей: Акира, Ливия, Райден.

## Что ещё НЕ подключено

Пока не подключено:

- Railway;
- Railway Volume;
- GPT Actions;
- живая игровая сессия.

То есть GitHub-коробка собрана, но сервер ещё не запущен.

## Следующий большой шаг

Нужно развернуть репозиторий на Railway.

После Railway появится ссылка вида:

```text
https://academy-1198-v3-production.up.railway.app
```

Тогда можно будет открыть:

```text
/health
/debug/volume
/openapi-actions.json
```

Если `/health` отвечает `success: true`, значит сервер живой.

## Что важно на Railway

Нужно создать Volume и подключить его к пути:

```text
/data
```

Переменные окружения:

```text
PROJECT_SLUG=academy-1198-v3
NOVELLA_RUNTIME_DATA_ROOT=/data
COMPACT_EVERY_TURNS=15
PUBLIC_BASE_URL=https://ВАША-ССЫЛКА.up.railway.app
```

## Как будет работать игра

1. GPT создаёт новую сессию.
2. Сессия сохраняется в `/data/sessions/...`.
3. Каждый ход GPT берёт контекст через API.
4. После сцены GPT сохраняет изменения:
   - state;
   - отношения;
   - знания;
   - open threads;
   - память персонажей.
5. Каждые 15 ходов делается сжатие памяти.

## Главное правило

Каждый новый ChatGPT-чат должен создавать новую runtime-сессию.

Нельзя использовать `session_id` из:
- старых чатов;
- тестов;
- документации;
- памяти ассистента;
- последней папки в `/data/sessions`.

Старую сессию можно продолжать только если пользователь явно дал конкретный `session_id` и попросил продолжить именно её.

## Что делать после Railway

После деплоя нужно прислать сюда публичную ссылку Railway. Тогда можно будет проверить:

- `/health`;
- `/debug/volume`;
- `/openapi-actions.json`;
- создание новой сессии;
- получение turn-contract;
- сохранение тестовой сцены.
